Customer Service Requests Analysis

The NYC 311 Data Analysis project focuses on analyzing customer service requests to understand common public complaints and service efficiency. The dataset contains thousands of records with details like complaint type, agency, location, and response time.
Using Python libraries such as Pandas, NumPy, Matplotlib, and Seaborn, the project performs data cleaning, exploration, and visualization to identify patterns and trends.

Methodology:
Data cleaning and handling missing values
Feature engineering (Request Closing Time)
Data visualization using graphs
Statistical testing using Kruskal-Wallis H Test

The analysis helps in finding:
Most frequent complaints,
Agencies handling requests,
Time taken to resolve issues,
Trends in complaints over time.
